What runs in the background

A handful of automations run quietly so the team doesn't have to remember things the system can remember. You don't need to touch any of these. If something here looks broken or you have an idea for a new one, message Nate.

Weekly Pipeline Report Fridays 4pm CT · #ec-guests

Posts pipeline health snapshot: Targeted count vs. threshold, gap coverage by topic, host balance (Matt / Josh / Either), what's new this week. Snapshot only, no action required from you.

Monday Guest Scoring Mondays 8am CT · #ec-guests

Tells Matt and Josh how many new Targeted guests need their fit score. Each link opens a blind scoring portal. About two minutes per host.

Idle Guest Watcher Daily 8am CT · #ec-guests

Flags any guest in Contacted or In Conversation status who's gone quiet beyond the configured threshold. Posts to #ec-guests so Nate or Emma can follow up.

Editorial calendar sync Continuous · Internal

Release dates from the editorial pipeline roll up to the shared Google Calendar so everyone sees the same publishing rhythm.

#intake → ClickUp (planned) Coming soon

Will auto-create ClickUp items from #intake submissions with URL parsed and beat tagged. Cuts the manual capture step.

Host-facing guest pipeline view (planned) Coming soon

The full guest CRM is currently admin-only. A planned automation will sync a read-only view to ClickUp or a shared Google Sheet so hosts can see the full pipeline without needing CRM access. Two-way sync so any updates flow back to the source of truth.

Got an automation idea? If you find yourself doing the same manual thing twice a week, it can probably be automated. Drop the idea in #general or DM Nate.
